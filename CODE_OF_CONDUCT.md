# Code of Conduct

This repository expects useful, respectful, and technically grounded behavior.

## Expected Behavior

- keep the discussion focused on code, docs, design, or system behavior
- bring evidence, examples, or a clear rationale
- write in a way that helps other engineers act on the feedback

## Unacceptable Behavior

- humiliation, insults, mockery, or harassment
- hostile or baiting comments
- non-constructive criticism with no practical value
- spam, noise, or repeated low-effort engagement

## Enforcement

The repository owner may edit, remove, ignore, lock, or report comments, issues, and pull requests that do not contribute in a useful way.
