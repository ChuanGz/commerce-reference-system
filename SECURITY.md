# Security Policy

## Scope

This repository is a public reference system. It is not a hosted product, but configuration leaks and unsafe defaults still matter.

## Reporting

- Do not post live secrets, exploit details, or private environment values in a public issue.
- Use GitHub private vulnerability reporting if available.
- Otherwise, contact the repository owner through GitHub first.

## What To Report

- hardcoded passwords, signing keys, or tokens
- unsafe Docker or appsettings defaults
- auth or authorization flaws that are visible in the checked-in code
- private infrastructure details that should not be public

## Response Model

- best effort only
- no SLA
- no guaranteed fix timeline
